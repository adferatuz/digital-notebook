POSIBILIDAD DE USAR SUPABASE COMO SERVIDOR Y BASE DE DATOS

El plan gratuito de Supabase es una opción viable y muy adecuada para la persistencia de datos de esta aplicación, especialmente considerando que no se cuenta con un backend propio y el objetivo es validar el producto antes de una inversión.

**Ventajas del Plan Gratuito de Supabase para este proyecto:**

*   **Tamaño de la Base de Datos:** El límite de 500 MB es más que suficiente. Con un estimado de 20-30 formularios EDAH al mes (incluyendo credenciales), el consumo anual sería de aproximadamente 4.3 MB, muy por debajo del límite.
*   **Transferencia de Datos (Egress):** El límite de 1 GB mensual es amplio. El bajo volumen de usuarios y formularios generaría una transferencia de datos mínima.
*   **Usuarios Activos Mensuales (MAUs):** El límite de 50,000 MAUs es extremadamente generoso. Con 1 psicóloga y ~30 tutores al mes, el uso estaría muy por debajo de este umbral.
*   **Almacenamiento de Archivos (Storage):** Si solo se almacenan datos de texto en la base de datos, el límite de 1 GB de almacenamiento de archivos no sería un factor.
*   **Invocaciones de Edge Functions:** El límite de 500,000 invocaciones mensuales es muy alto. Las operaciones de guardar/leer credenciales y formularios generarían solo unas pocas docenas o cientos de invocaciones al mes.

**Conclusión:** El plan gratuito de Supabase es robusto y generoso para el caso de uso actual, permitiendo operar la aplicación sin costos iniciales y escalando de forma fluida si el proyecto crece.
